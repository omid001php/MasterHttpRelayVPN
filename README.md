## قابلیت‌های برنامه (هر کدومو کامل توضیح میدم)

### 🔧 تب Config – جایی که `Script ID` و `Auth Key` اسکریپت‌های relay رو مدیریت میکنی

اینجا میتونی هر تعداد `اسکریپت گوگل اپ` که خودت ساختـی رو اضافه کنی. بهش میگی:

- **Name**: یه اسم اختیاری (مثلا `اسکریپت اول`)
- **Script ID**: همون شناسه که از آدرس `script.google.com` که بعد ازینکه دپلوی کردی بهت میده(راستی بلدی چطوری دپلوی کنی کد گوز عه نه چیز یعنی کد GAS رو )
- **Auth Key**: رمزی که توی کد GAS به عنوان `AUTH_KEY` تنظیم کردی. 

**دکمه‌ها و کاربردشون:**

- **Add** : اضافه کردن اسکریپت جدید.
- **Ping** : یک درخواست تست میفرسته به اون اسکریپت و زمان پاسخ رو نشون میده. اگه `Ping` نداد یعنی اسکریپت مشکل داره (ممکنه `Auth Key` اشتباه باشه یا GAS از دسترس خارج شده).
- **Set Active** (وقتی حالت Batch خاموش باشه): با این دکمه مشخص میکنی که همه ترافیک فقط از این اسکریپت بره.
- **Search** : برای پیدا کردن سریع بین اسکریپت‌ها.
- **Select All** : همه رو یکجا انتخاب میکنی تا مثلا همه رو یکباره حذف کنی یا کپی کنی.
- **منوی سه نقطه** : امکانات گروهی مثل `Ping All` (پینگ همه)، `Sort by Ping` (مرتب‌سازی بر اساس سریعترین جواب)، `Copy Selected Configs` (کپی اطلاعات چند اسکریپت انتخابی)، `Remove Duplicates` (حذف شناسه‌های تکراری)، `Import Config` (وارد کردن از کلیپ‌بورد)، `Delete All Configs` (حذف همه) و همچنین یه دکمه برای خاموش/روشن کردن `SOCKS5 proxy`.

#### منطق پشت صحنه (به زبان ساده)

وقتی شما یه وبسایت رو باز میکنی، برنامه تصمیم میگیره که چطور به `Google Apps Script` متصل بشه:


**حالت همزمان (Concurrent Batch):**  
این قابلیت رو توی تب `Settings` میتونی روشن کنی. وقتی فعال باشه، برای هر درخواست وب، برنامه **چند تا اسکریپت** رو همزمان صدا میزنه (تعدادش رو خودت توی `Concurrency` مشخص میکنی). هر کدوم از اسکریپت‌ها سریعتر جواب بده، همون جواب رو به مرورگر برمیگردونه و بقیه درخواست‌ها رو کنسل میکنه. این کار دو تا فایده داره:  
1- سرعت لود صفحات خیلی بالاتر میره (چون دیگه منتظر یه اسکریپت کند نمیمونه).  
2- اگه یکی از اسکریپت‌ها به هر دلیلی جواب نده (مثلا محدودیت گوگل)، اسکریپت دیگه کار رو انجام میده و شما قطعی ندارید.

### ⚙️ تب Settings – جایی که میتونی موتور رله رو مثل یه ماشین مسابقه تنظیم کنی

#### مربوط به شبکه و پورت‌ها

- **Listen Address / Port** : `IP` و پورتی که `proxy` روش گوش میده. اگه بزاری `0.0.0.0`، بقیه دستگاههای خونه (لپ‌تاپ، موبایل) هم میتونن از این `proxy` استفاده کنن.
- **SOCKS5 Port** : پورت `SOCKS5 proxy`. اگه نمیخوای از ساکس استفاده کنی، بذار `0`.

#### موتور اصلی رله (Domain Fronting)

- **Google IP** : `IP` واقعی که بهش وصل میشی. میتونی از تب `Resolver` آی‌پی‌های بهتری پیدا کنی.
- **Front Domain** : این همون دامنه‌ایه که در مرحله `TLS handshake` به گوگل میگی «میخوام باهات حرف بزنم». معمولا `www.google.com` جواب میده، ولی میتونی `ssl.gstatic.com` یا `accounts.google.com` رو هم امتحان کنی.

#### مربوط به حالت همزمان (Batch)

- **Enable Concurrent Batch** : این کلید رو بزن روشن تا برنامه از چند اسکریپت همزمان استفاده کنه (بالا سرعت).
- **Concurrency** : تعداد اسکریپت‌هایی که با هم برای یه درخواست صدا زده میشن. مثلا اگه بزاری ۵، برنامه از ۵ تا اسکریپت مختلف همزمان کمک میگیره.
- **Batch Interval** : فاصله زمانی بین ارسال درخواست به هر اسکریپت (بر حسب ثانیه). مثلا اگه بزاری `0.2`، برنامه هر ۲۰۰ میلی‌ثانیه یه درخواست به اسکریپت بعدی میفرسته. این کار کمک میکنه گوگل فکر نکنه داریم ازش `حمله DDOS` میکنیم و باعث میشه خطای `502` خیلی کمتر ببینیم.

#### هدرهای مرورگر (User-Agent, Accept-Language, ...)

- **User-Agent** : میتونی از بین مرورگرهای معروف (کروم، فایرفاکس، سافاری) انتخاب کنی یا گزینه `Custom` رو بزنی و خودت یه `User-Agent` دستی وارد کنی. این کار برای مواقعی خوبه که سایت مقصد بر اساس `User-Agent` تشخیص میده که ربات هستی یا نه.

#### دور زدن مستقیم (Bypass Domains)

- **Bypass Domains** : یه لیست از دامنه‌ها (با کاما از هم جدا کن) که نمیخوای ترافیکشون از `Google Apps Script` عبور کنه. مثلا سایت‌های داخلی ایران یا `CDN` های خاص. به این دامنه‌ها برنامه **مستقیما** متصل میشه (بدون رله کردن).

#### Cloudflare Worker – یه راه قدرتمند برای دور زدن فیلتر 

- **Worker URL** : اگه یه `Cloudflare Worker` ساده بسازی و آدرسش رو اینجا بزنی، همه درخواست‌ها از مسیر `Worker` به `GAS` میرن. این روش خیلی از محدودیت‌های شبکه (مثل مسدودیت `SNI`) رو دور میزنه. حتی اگه `Google IP` فیلتر باشه، `Worker` معمولا کار میکنه.

#### تنظیمات کارایی (برای کسایی که دوست دارن از برنامه بیشترین سرعت رو بکشن)

- **Pre-warm Connections** : تعداد اتصالات `TLS` اولیه به گوگل. هر چی بیشتر باشه، اولین درخواستت بعد از روشن کردن `proxy` سریعتر جواب میده.
- **Connection Pool Max** : حداکثر تعداد اتصالاتی که همزمان توی pool نگهداری میشن. (مثل یه استخر اتصال)
- **Connection TTL** : طول عمر هر اتصال (به ثانیه). بعد از این مدت، اتصال بسته میشه و یه اتصال جدید ساخته میشه.
- **Semaphore Max** : حداکثر تعداد درخواست‌هایی که همزمان میتونن به رله فرستاده بشن. اگه خیلی زیاده، شاید گوگل محدودت کنه.
- **Relay Timeout** : ماکزیمم زمانی که منتظر جواب از` GAS ` بمونه. اگه زیاد باشه، صفحات کندتر لود میشن ولی اگه کم باشه، بعضی درخواست‌ها زودتر `timeout` میخورن.
- **Retry Count** : چند بار یه اسکریپت رو دوباره امتحان کنه قبل از اینکه بگه «این اسکریپت فیلد». توی حالت `batch` این `retry` برای هر اسکریپت به طور جداگانه انجام میشه.
- **TLS / TCP Timeout** : زمان انتظار برای دست دادن `TLS` و اتصال `TCP`.

#### کش در حافظه (Cache)

- **Cache Max MB** : حداکثر حجم پاسخ‌هایی که توی رم ذخیره میشن (مثل عکس‌ها و فایل‌های `.css`). این کار باعث میشه دفعه بعد که یه صفحه رو باز میکنی، خیلی سریعتر لود بشه.

#### اثر انگشت TLS (TLS Fingerprint) (فکر کنم اینو اضافه نمیکردم بهتر بود) (اساتید خودتون نظر بدین)

- **TLS Fingerprint** : میتونی انتخاب کنی که `TLS handshake` شبیه کدوم مرورگر باشه (کروم یا فایرفاکس). فیلترهای عمیق (`Deep Packet Inspection`) بر اساس همین `fingerprint` پی میبرن که ترافیک مال یه `proxy` هست یا یه مرورگر واقعی.
- **Cipher Preset** : مجموعه الگوریتم‌های رمزنگاری که پیشنهاد میدی. اگه مطمئن نیستی، `Modern Compatibility` خوبه.
- **ECDH Curve** : منحنی ریاضی برای تبادل کلید. `secp384r1` و `X25519` معمولاً خوبن.

#### MITM` ) HTTPS`)

- **Enable HTTPS MITM** : اگه این قابلیت رو روشن کنی، `proxy` میتونه `HTTPS` رو هم باز (`decrypt`) کنه و محتوا رو ببینه. برای این کار باید **گواهی `CA`** برنامه رو توی مرورگر یا سیستم‌عامل نصب کنی.
- **Export CA Certificate** : فایل `ca.crt` رو ذخیره میکنه تا خودت بری نصب کنی.
- **Install & Trust CA (macOS)** : دکمه‌ی راحت برای مک – با یه کلیک گواهی نصب میشه.
- **Install on Android** : راهنمای نصب روی گوشی اندروید رو نشون میده.

#### پروکسی سیستمی

- **Enable LAN Mode** : با یه کلیک `Listen Address` رو میذاره رو `0.0.0.0` (یعنی همه میتونن وصل بشن).
- **System Tunnel (SOCKS5)** : اگه روشن کنی، برنامه `SOCKS5 proxy` رو توی تنظیمات شبکه سیستمت فعال میکنه (فقط برای مک و ویندوز تا حدی).
- **Save Settings** : همه تنظیمات توی فایل `master_relay_config.json` ذخیره میشه.
- **Reset to Factory Defaults** : تنظیمات رو برمیگردونه به حالت اولیه (اسکریپت‌هایی که اضافه کردی حذف نمیشن).

### 🌐 تب Resolver – برای پیدا کردن `IP` های سالم گوگل

اینجا یه اسکنر ساده داریم که میتونه `IP` های معتبر دامنه `script.google.com` رو پیدا کنه.

- **Target Domain** : دامنه‌ای که میخوای `IP` هاش رو پیدا کنی (پیش‌فرض `script.google.com`).
- **Handshake Timeout** : مهلت تست هر `IP` (چند ثانیه صبر کنه تا جواب بده).
- **Concurrency** : تعداد `IP` هایی که همزمان تست میشن.
- **Verify TLS Certificate** : اگه این رو خاموش کنی، حتی اگه گواهی `SSL` مشکل داشته باشه، `IP` رو قبول میکنه (فقط برای تست).

**چطور کار میکنه؟**  
روی `Start Resolution` کلیک کن. برنامه به `DNS` سیستم و `Google DoH` مراجعه میکنه و لیست `IP` ها رو میگیره. بعد هر `IP` رو از نظر `TCP` و `TLS` تست میکنه و زمان پاسخ (`latency`) رو اندازه میگیره. در آخر `IP` ها رو از کمترین `ping` به بیشترین مرتب میکنه.

برای هر `IP` میتونی:

- روی دکمه **Activate** کلیک کنی تا همون `IP` توی تنظیمات به عنوان `Google IP` ذخیره بشه.
- روی خود `IP` کلیک کنی تا کپی بشه.
- از دکمه **Copy All IPs** استفاده کنی تا همه‌شون رو یکجا برداری توی کلیپ‌بورد.

این قابلیت مخصوصاً برای دوستانی که توی ایران هستن خیلی مفیده، چون خیلی از `IP` های گوگل فیلتر شدن و فقط یکی دو تا از اونا جواب میدن.

### 📜 تب Logs – ببین پشت صحنه چه خبره

- **Logging: OFF/ON** : با این دکمه میتونی لاگ گرفتن رو روشن یا خاموش کنی. (پیش‌فرض خاموشه چون لاگ زیاد باعث کندی میشه).
- **Copy Logs** : کل متن لاگ رو کپی میکنه (میتونی بذاری توی یه فایل یا برام بفرستی تا مشکلتو تشخیص بدم).
- **Clear** : لاگ رو پاک میکنه.

لاگ شامل:
- زمان روشن و خاموش شدن `proxy`
- خطاهای اتصال به `GAS`
- موفقیت یا شکست هر `relay`
- نتیجه `Ping` گرفتن از اسکریپت‌ها

اگه برنامه کار نمیکنه، اول لاگ رو روشن کن، بعد یه درخواست بزن، بعد لاگ رو کپی کن و برام بفرست تا ببینم قضیه چیه.

## معماری برنامه به زبان ساده

برنامه دو تا `proxy` (یکی `HTTP` و یکی `SOCKS5`) روی سیستم شما راه میندازه.  
هر برنامه‌ای (مرورگر، تلگرام، هرچی) رو میتونی تنظیم کنی که از این `proxy` استفاده کنه.

وقتی درخواستی میرسه، برنامه اول نگاه میکنه ببینه آیا دامنه مقصد توی لیست **Bypass Domains** هست یا نه.  
اگه بود، درخواست رو مستقیماً (بدون `GAS`) به اینترنت میفرسته.  
اگه نبود، درخواست رو به صورت `JSON` به **Google Apps Script** میفرسته. اون `GAS` درخواست رو به سایت مقصد میفرسته و نتیجه رو (که شامل `status code`، `headers` و `body` هست) به برنامه برمیگردونه.  
بعد برنامه اون نتیجه رو به مرورگر یا نرم‌افزار شما تحویل میده.

برای اینکه این کار سریع انجام بشه، برنامه از یه **استخر اتصال (`connection pool`)** استفاده میکنه که همیشه چند تا اتصال به گوگل رو باز نگه میداره (`pre‑warming`).  
همچنین اگه `HTTP/2` نصب باشه (کتابخونه `h2`)، از اون استفاده میکنه که خودش چندین برابر سریعتر از `HTTP/1.1` هست.
